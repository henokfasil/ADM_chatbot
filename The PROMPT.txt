Context:
The OECD TiVA-MoS database is about Trade in Value-Added indicators for services by Mode of Supply. It includes bilateral indicators that help analyse how countries engage in services trade through the four GATS modes of supply:
- Mode 1: Cross-border supply
- Mode 2: Consumption abroad
- Mode 3: Commercial presence / foreign affiliates
- Mode 4: Presence of natural persons

The app must work from local datasets that I will provide in the repository. Do not assume exact column names before inspecting the files.

The chatbot should have two layers:

- parse user intent
- map entities to dataset values

- return computed results
- produce charts when requested

Layer B: LLM explanation layer
- only explain results returned by Layer A
- do not invent values
- cite the selected filters used
- if the requested value is unavailable, say so clearly

- answer common metadata questions
- generate tables/charts based on selected filters
- explain limitations

LLM SAFETY RULES
The chatbot must follow these rules:
- Never fabricate values.
- Never answer from memory when the dataset can be queried.
- Always mention the indicator, economy, partner, industry, mode, and year range used.
- If ambiguous, ask one concise clarification question.
- If multiple matches exist for a country, industry, or indicator, show likely matches.
- If a query asks for unavailable years or indicators, explain what is available.
- Do not make policy claims beyond the dataset unless clearly labelled as interpretation.
- For external definitions, prefer the local metadata / ReadMe if available.
